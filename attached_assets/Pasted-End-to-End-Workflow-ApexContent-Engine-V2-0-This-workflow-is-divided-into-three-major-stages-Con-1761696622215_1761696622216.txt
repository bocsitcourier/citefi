End-to-End Workflow: ApexContent Engine (V2.0)
This workflow is divided into three major stages: Configuration (User Input), Processing (Asynchronous Pipeline), and Distribution (CMS/Export).

I. 🎯 Stage 1: Job Configuration & Title Selection (Frontend/SSR)
This stage occurs synchronously on the user's browser, culminating in the submission of a bulk job to the backend queue.

Step	User Action	System Action (API Call)	Data Stored/Key Outcome
1.1 Login & Setup	User signs in via /signin page.	NextAuth.js verifies credentials against the users table, establishes session.	User session and permissions established.
1.2 Input Topic	User enters Core Topic, Target Word Count, and other constraints (Audience, Tone).	N/A	Local state variables set.
1.3 Generate Title Pool	User clicks "Generate Title Pool."	POST /api/jobs/title-pool (Node.js/Gemini API)	50 unique titles returned. Primary SEO set generated. job_batches.title_pool_json created in DB.
1.4 Title Selection	User reviews the 50 titles and selects 1 to 50 for the batch.	N/A	selectedTitles array finalized on frontend.
1.5 Batch Submission	User confirms the batch size and clicks "Start Bulk Job."	POST /api/jobs/batch-submit (Node.js/BullMQ Queue)	A new job_batches record is created (Status: PENDING). Job is added to the BullMQ queue.

II. 🧠 Stage 2: Asynchronous Bulk Processing (Backend/BullMQ Worker)
The BullMQ Worker picks up the job and iterates through every selected title. This loop happens entirely in the background.

Step	Worker Process Action	External API / Tool Used	DB Interaction / Status Update
2.1 Worker Start	Worker pulls job_batches record from the queue.	BullMQ/Redis	job_batches.status set to RUNNING.
2.2 Article Loop Start	Worker begins iteration for each selected title (Max 5 concurrent).	N/A	A new articles record is created (Status: PENDING).
2.3 Content & Metadata	Function 1: generateArticleCore() generates text and all unique metadata (slug, meta, keywords).	Gemini 2.5 Pro (Structured JSON Output)	articles record updated with article_text, slug, and all SEO fields.
2.4 Image Assets	Function 2: generateAssetsAndStore() generates 5 images and optimizes them.	Imagen / Sharp (Node.js Library)	5 records created in article_assets with DO Spaces URLs.
2.5 Social Posts	Function 3: generateSocialPosts() creates platform-specific captions/hashtags.	Gemini 2.5 Flash	3 records created in article_social_posts.
2.6 Quality Assurance	Function 4: checkContentQuality() runs plagiarism, readability, and schema generation.	External Plagiarism API / GPT-4	articles record updated with plagiarism_score, readability_score, and schema_json_ld.
2.7 Final Formatting	Function 5: finalizeContent() injects internal links, image tags, and formats to final HTML.	GPT-4 (HTML Injection)	articles.final_html_content field is fully populated. articles.article_status set to COMPLETE.
2.8 Loop End	Worker fires a WebSocket event and checks for article failure.	WebSockets	Real-time status update sent to the client dashboard. If articles failed, job_batches.status is set to PARTIAL_COMPLETE.
2.9 Job Completion	All articles processed. Worker executes cleanup and notification logic.	SendGrid (Email)	User receives email notification. job_batches.status set to COMPLETE.

III. 🌐 Stage 3: Review, Edit, & Distribution (Frontend/CMS Actions)
The user interacts with the generated content in the secure dashboard.

Step	User Action	System Action (API Call)	Data Stored/Key Outcome
3.1 Review/Edit	User navigates to the Article Editor, reviews content, and makes edits (HTML/WYSIWYG).	GET /api/content/[id] loads HTML and metadata.	N/A
3.2 Save Edits	User clicks "Save."	PUT /api/content/[id]/save (Node.js/RBAC Check)	articles.final_html_content record is overwritten.
3.3 Batch Export	User selects multiple articles and clicks "Export."	GET /api/export/batch/[id]	User downloads a ZIP file (e.g., Markdown/HTML files) and/or a CSV metadata report.
3.4 Direct Publish	User selects an article and clicks "Publish to WordPress."	POST /api/publish/article/[id] (Node.js/CMS API)	Article pushed to the external CMS. published_posts record created with external_url and PUBLISHED status.
3.5 Social Push	User clicks "Schedule Post" on the Social tab.	POST /api/social/schedule (Node.js/Buffer API)	Content syndicated to the chosen social media platform.
3.6 Deletion	User clicks "Delete Article."	DELETE /api/content/[id]/delete (Node.js/RBAC Check)	Hard deletion from articles, article_assets, and DigitalOcean Spaces (asset cleanup).
