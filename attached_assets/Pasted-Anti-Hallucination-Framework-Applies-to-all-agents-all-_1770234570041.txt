Anti‑Hallucination Framework
Applies to all agents, all content types.

Rule 0 – Transformation Only
Agents may only transform verified inputs.
They may never introduce new facts.
This is enforced in architecture, not prompts.

1. Single Source of Truth
Fact Store (Immutable)
All agents read from one canonical fact store.

json
{
  "fact_id": "f_1029",
  "statement": "Company provides same-day courier services",
  "source": "company_website",
  "verified_at": "2026-02-01",
  "confidence": 0.94,
  "expires_at": null
}
Hard constraints:

Agents cannot write to this store.

Agents cannot edit existing facts.

Agents cannot invent substitute facts.

If it’s not in the store, it does not exist.

2. Agent Input Contract
Every agent receives structured input only.

json
{
  "facts": ["f_1029", "f_2041"],
  "allowed_operations": ["rewrite", "summarize", "format", "visualize"],
  "forbidden_operations": ["assume", "infer_missing_facts", "browse"],
  "confidence_floor": 0.7
}
If an agent attempts an operation outside allowed_operations, execution halts.

3. Evidence Binding (Sentence‑Level)
Every generated element must bind to evidence.

Internal bindings:

Sentence → fact_id[]

Visual element → concept_id[]

Claim → fact_id[]

Example:

Output: “We offer same‑day courier services across the region.”

Internal requirement: sentence_001 → ["f_1029"]

If a sentence cannot be bound to one or more facts, it is dropped automatically.

4. Claim Classification Layer
All candidate statements are classified before they leave the system.

Claim Types
Direct fact → allowed (must cite)

Rephrase of known fact → allowed

Derived insight (e.g., aggregation/combination of facts) → allowed (must cite + confidence score)

Assumption → rejected

Guess → rejected

Industry generalization → rejected

Anything rejected is deleted, not “softened” or rewritten.

5. Confidence Gating
Agents cannot emit claims below the configured confidence floor.

Example:

json
{
  "claim": "High-volume logistics provider",
  "confidence": 0.42
}
Result: rejected – below confidence floor.

Confidence gating applies uniformly to:

Long‑form content (blogs, whitepapers)

Short‑form (social posts, ad copy)

Scripts (video, audio)

Captions and alt text

Feedback or “AI coach” responses

6. No Cross‑Agent Fact Leakage
Agents never “learn facts” from each other’s outputs.

Learning agents may only ingest:

Metrics (CTR, time on page, reply rates, etc.)

Engagement patterns

Outcome signals (conversion, retention, etc.)

They may not ingest:

Generated sentences or paragraphs

Claims or “insights” in text form

Summaries of other agents’ outputs

This prevents hallucinated content from becoming “truth” via repetition.

7. Gap Declaration (No Guessing)
If required data is missing, the only valid response is a gap report:

json
{
  "status": "INSUFFICIENT_DATA",
  "missing": ["service_area", "fleet_size"]
}
No soft‑filling.
No hedging (“likely,” “probably,” “typically”).
No fallbacks to generic industry boilerplate.

8. Visual & Video Safeguards
Visual agents are constrained to concepts, not business claims.

They receive only:

json
{
  "concepts": ["secure_delivery"],
  "mood": "trust",
  "style": "clean",
  "forbidden_elements": ["logos", "faces", "text"]
}
They do not receive:

Company names

Operational promises

Performance claims

Captions and overlays are generated by a separate text agent and re‑validated against the fact store before being attached to the asset.

9. Learning Layer Safeguards
Learning is purely behavioral, not factual.

Allowed learning:

Format effectiveness (e.g., list vs. narrative)

Length optimization

Tone preference within allowed styles

CTA placement and structure

Forbidden learning:

New domain facts

Industry “rules of thumb”

User intent inference beyond observed signals

Pattern‑based “knowledge” (e.g., “most logistics firms do X, so we say X”)

Learning can only change how content is presented, never what is asserted.

10. System‑Level Kill Switches
The system aborts, rather than retries, when safety conditions fail.

Automatic abort triggers:

Claim without fact binding

Claim confidence < configured threshold

Referenced fact is expired

Conflicting sources for the same fact with no resolution

Output contains unbound noun phrases that look like factual entities (e.g., new locations, products, metrics)

Abort means: stop, log, surface error. No silent degradation, no speculative retry.

11. Audit Trail
Every output artifact stores a minimal, immutable audit record:

json
{
  "output_id": "uuid",
  "facts_used": ["f_1029"],
  "confidence_avg": 0.91,
  "agents_involved": ["generator", "reviewer"],
  "timestamp": "2026-02-04T14:47:00Z"
}
This guarantees:

Traceability (what facts drove this?)

Explainability (why did we say this?)

Defensibility (regulatory and enterprise compliance)

12. Why This Works
Hallucinations emerge when:

The model fills gaps with “likely” content.

Agents learn from generated text instead of ground truth.

Creative generation and factual assertion share the same channel.

This framework cuts all three:

No gaps: missing data is surfaced, not guessed.

No self‑reinforcement: generated text never becomes a fact source.

No blended modes: creativity can only rearrange verified content.

You’re not “trusting” the model to behave.
You’re designing the system so that, structurally, it cannot hallucinate facts.
