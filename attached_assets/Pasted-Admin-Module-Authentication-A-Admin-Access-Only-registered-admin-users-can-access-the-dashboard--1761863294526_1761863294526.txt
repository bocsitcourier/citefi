Admin Module & Authentication
A. Admin Access

Only registered admin users can access the dashboard and control panels.

Standard web app login flow:

Email + Password

Multi-factor authentication (optional, recommended for high-security deployments)

JWT session tokens or secure cookies for authentication

Admin role stored in users.role = 'admin'

API Endpoints:

POST /api/auth/login
Body: { email, password }
Response: { token, role }

GET /api/admin/dashboard
Headers: Authorization: Bearer <token>
Response: { stats, pending_jobs, recent_errors, seo_reports }

POST /api/auth/logout
Headers: Authorization: Bearer <token>
Response: { status: "success" }

B. Admin Dashboard Features

1. Real-time Overview

Jobs in progress, pending, or failed

Token usage & Cost_Per_Rank metrics

Refresh suggestion badges for underperforming articles

Geo-Accuracy score per article

2. Article & Batch Management

View all batches/articles regardless of submitting user

Approve, edit, refresh, or delete articles

Retry failed jobs

Export reports

3. Error Monitoring

Centralized error log panel

Tracks:

Gemini API failures

GPT-4 API failures

Media upload errors

Schema validation failures

Token/cost anomalies

error_logs table updated in real-time

4. Additional Admin Tools

Search and filter articles by status, geo-score, or topic

Trigger manual content refresh jobs

Force re-generation for any article/batch

C. Database Additions for Admin

1. error_logs (expanded for admin visibility)

Column	Type	Constraints	Description
id	SERIAL	PK	Error ID
article_id	INT	FK → articles.id	Article reference
batch_id	INT	FK → job_batches.id	Batch reference (optional)
error_type	VARCHAR(50)	NOT NULL	Gemini, GPT, Schema, MediaUpload, AdminAction
message	TEXT	NOT NULL	Detailed message
resolved	BOOLEAN	DEFAULT FALSE	Whether admin marked error resolved
created_at	TIMESTAMP	DEFAULT NOW()	Timestamp
resolved_at	TIMESTAMP	NULL	Timestamp when resolved

2. users (already includes role column)

Only role = 'admin' can access /api/admin/* endpoints

E. Security & Access

All admin APIs require JWT or session auth

Admin actions logged in admin_action_logs table (optional but recommended)

Sensitive data (API keys, user info) never exposed to frontend

F. Async & Job Integration

Admin can manually retry failed jobs → pushes to async BullMQ job queue

Admin-triggered refresh jobs follow same AI pipeline (Gemini → GPT-4 → SEO/Geo optimization → DB + Spaces)
