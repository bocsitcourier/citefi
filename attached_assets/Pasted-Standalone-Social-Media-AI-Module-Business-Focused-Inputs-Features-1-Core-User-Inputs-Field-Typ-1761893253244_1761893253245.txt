Standalone Social Media AI Module – Business-Focused Inputs & Features
1. Core User Inputs
Field	Type	Required	Description
topic	Text	✅	Main topic of the article or post
title	Text	✅	Article title or post headline
location	Text	✅	City, neighborhood, or landmark for geo-targeting
tone	Dropdown	✅	Professional, Authoritative, Friendly-Professional, Insightful, Executive
mood	Dropdown	Optional	Motivational, Analytical, Informative, Persuasive, Neutral
industry	Dropdown	Optional	E.g., Logistics, Finance, Healthcare, Retail, Tech
socialHandles	Multi-select	✅	User’s accounts on X, Facebook, Instagram, LinkedIn, Pinterest
numberOfPosts	Number	Optional	How many AI-generated posts to create (default 3 per platform)
imagePreference	Radio/Checkbox	Optional	Generate hero image / platform-specific image / none
autoShare	Checkbox	Optional	Automatically share generated posts to selected accounts

Notes:

The moods and tones are now tailored to business audiences, focusing on trust, professionalism, and clarity.

Users can still select multiple handles and decide if posts auto-share or not.

2. AI Post Generation Workflow

User Input → Smart AI Prompt Generator

Uses all business-oriented inputs (topic, title, location, tone, mood, industry)

Integrates SEO & GEO keyword analysis

Generates hashtags, emojis, and hyperlinking instructions for business audiences

BLUF principle: posts lead with the main insight first

Content Generation

Produces multiple professional post variations per platform

Text formatted for platform-specific character limits

Emojis added subtly to complement business tone

Hero/platform-optimized images generated concurrently

SEO & GEO Optimization

Incorporate location-specific business keywords

Geo-targeted hashtags (#BostonLogistics, #NYCFinance)

Hyperlink hashtags to user email or landing pages

Semantic relevance maintained to location, topic, and business context

Editor & Preview

Preview posts per platform

Edit text, hashtags, hyperlinks, images, tone, and mood

Delete individual posts or entire batch

Live SEO/GEO optimization scoring

Auto-share / Manual Posting

Posts queued for auto-share to selected platforms

Dashboard updates with status (shared, pending, failed)

Logs stored for auditing and error tracking

3. Backend API Updates
Endpoint	Method	Description	Input	Output
/api/social-posts/generate	POST	Generate business-focused posts based on user inputs	topic, title, location, tone, mood, industry, socialHandles[], numberOfPosts, imagePreference	posts[] { platform, text, hashtags[], imageUrl, geoTags }
/api/social-posts/edit/[id]	PUT	Edit post content	postId, updatedText, updatedHashtags, updatedImage, updatedTone, updatedMood	post object
/api/social-posts/delete/[id]	DELETE	Remove a post	postId	success/failure
/api/social-posts/share	POST	Share posts to selected accounts	postId[], socialHandles[]	status[] per handle
/api/social-posts/progress	GET	Get generation progress	batchId	{ completed, total, status, errors }
4. ERD Update

Tables:

social_posts (updated for business tone and mood)

Column	Type	Description
id	SERIAL PRIMARY KEY	Post ID
topic	VARCHAR	User-defined topic
title	VARCHAR	User-defined title
location	VARCHAR	City, neighborhood, or landmark
tone	VARCHAR	Professional tone selected
mood	VARCHAR	Business-focused mood
industry	VARCHAR	Optional industry
platform	VARCHAR	X, Facebook, Instagram, etc.
text	TEXT	Generated post content
hashtags	JSONB	Array of hashtags
image_url	TEXT	Generated hero/platform-specific image
geo_tags	JSONB	City, neighborhood
seo_keywords	JSONB	Primary + secondary keywords
status	VARCHAR	pending, shared, failed
created_at	TIMESTAMP	Default NOW()

social_post_logs (track input + output)

Column	Type	Description
id	SERIAL PRIMARY KEY	Log ID
post_id	INT	Reference to social_posts
action	VARCHAR	generated, edited, shared
input_data	JSONB	Full user input payload
output_data	JSONB	Generated post details
errors	JSONB	Error messages if any
timestamp	TIMESTAMP	Default NOW()

6. Additional Developer Notes

Concurrency: Use Promise.all for generating multiple posts and images simultaneously.

SEO & GEO: Business and location-specific keywords incorporated.

Error Handling: Failures in image generation or API calls don’t halt the batch.

Admin Logs: Track all user inputs, outputs, errors, and completion status.

Platform Optimization: Enforce character limits, image dimensions, and subtle emoji placement suitable for professional audiences.
