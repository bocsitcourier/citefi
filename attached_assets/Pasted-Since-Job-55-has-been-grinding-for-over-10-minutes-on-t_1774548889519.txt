🧯 The "Clean-Start" Recovery Script (scripts/kill-and-restart.ts)

TypeScript
import { exec } from 'child_process';
import { promises as fs } from 'fs';
import { db } from '../lib/db';
import { articles } from '../lib/schema';
import { eq } from 'drizzle-orm';

async function rescueJob(jobId: string) {
  console.log(`🛡️ Rescuing Job ${jobId}...`);

  // 1. KILL the zombie FFmpeg process
  // This looks for any ffmpeg process that mentions your jobId in the path
  exec(`pkill -f "video-${jobId}"`, (err) => {
    if (err) {
      console.log(`ℹ️ No active FFmpeg process found for ${jobId} (already dead).`);
    } else {
      console.log(`💥 Successfully killed zombie FFmpeg for Job ${jobId}.`);
    }
  });

  // 2. WIPE the corrupted workspace
  const workDir = `/tmp/video-${jobId}`;
  try {
    await fs.rm(workDir, { recursive: true, force: true });
    console.log(`🧹 Workspace ${workDir} has been purged.`);
  } catch (e) {
    console.log("ℹ️ Workspace already empty.");
  }

  // 3. RE-QUEUE in the Database
  // This triggers the new Single-Pass logic on the next worker cycle
  await db.update(articles)
    .set({ 
      status: 'QUEUED', 
      progress_percent: 0,
      current_step_label: '🔄 Restarted with Single-Pass Platinum Logic',
      validation_logs: [] 
    })
    .where(eq(articles.id, jobId));

  console.log(`✅ Job ${jobId} is back in the queue. It will now run at 'ultrafast' speed.`);
}

// Get ID from command line: npx ts-node scripts/kill-and-restart.ts 55
const targetId = process.argv[2];
if (!targetId) {
  console.error("❌ Please provide a Job ID (e.g., 55)");
} else {
  rescueJob(targetId);
}

🎥 The "Veo" Perspective
If you eventually migrate to Veo for your video generation, the "Clean-Start" logic is actually simpler because there is no local /tmp mess to clean up.

FFmpeg: You have to manually kill local processes and delete files (what we just did).

Veo: You simply stop polling the Google API and "Retry" the request. Since the rendering happens on Google's infrastructure, a "Clean-Start" is just a new API call with a fresh request_id.
