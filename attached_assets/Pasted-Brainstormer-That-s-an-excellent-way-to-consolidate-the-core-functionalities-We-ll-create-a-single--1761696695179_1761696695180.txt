🚀 VIII. Media Handling & Advanced Editor Workflow
The core challenge is securely uploading large media files (audio, video) to DigitalOcean Spaces and ensuring the Rich Text Editor (RTE) places the resulting public URL correctly into the article's HTML.

A. 📝 Rich Text Editor Selection & Integration
We need a powerful, production-grade RTE that supports media integration and HTML editing.

Feature	Requirement	Recommended RTE Libraries
Rich Text Editing	WYSIWYG for standard users.	TinyMCE or CKEditor 5 (Both offer robust media management and Node.js SDKs).
HTML Source Editing	Dedicated button/tab to edit the raw HTML, including syntax highlighting.	Both TinyMCE and CKEditor offer Enhanced Code Editor plugins.
Media Handling	Toolbar buttons for Upload Image, Upload Audio, Upload Video, and Embed Media (for YouTube/Vimeo iframes).	Both offer dedicated media plugins that intercept the upload request.
B. ⬆️ Backend Media Upload Workflow (Node.js/Express)
The current POST /api/assets/upload endpoint must be generalized to handle multiple media types. This is the sequence the backend must follow when the user clicks the "Upload" button inside the RTE.

Step	Endpoint & Method	Node.js Library Used	Description & DigitalOcean Action
1. Client Upload	POST /api/media/upload	Multer (for handling multipart/form-data)	The RTE sends the file (image/audio/video) to this endpoint. Multer manages the stream.
2. Server Validation	N/A	file-type or similar	Validate file size, mime-type (e.g., video/mp4, audio/mpeg), and run security checks before storage.
3. Processing & Encoding	N/A	Sharp (Images), FFmpeg (Audio/Video)	Images: Convert to WebP and optimize (as planned). Video/Audio: This is complex. For large files, stream it directly to Spaces (see step 4) or use a cloud service (e.g., Cloudinary, as mentioned in search results) to handle chunking, format conversion, and streaming, reducing load on the DO App Platform.
4. Storage	N/A	AWS SDK/DigitalOcean Spaces	The processed file is streamed and saved to the appropriate bucket on DigitalOcean Spaces. A public, secure URL is generated (e.g., https://assets.kamoo.com/video/batch_id/my-video.mp4).
5. Database Record	N/A	PostgreSQL ORM	A record is inserted into the article_assets table: type: 'video', storage_url: '...', article_id: [id].
6. Response to Editor	POST /api/media/upload returns a JSON object.	Express res.json()	The response must contain the final public URL. The RTE uses this URL to automatically insert the correct HTML tag (<img src="...">, <audio src="...">, or <video src="...">) into the content.
C. 🔗 Embedding External Media (iFrames)
For platforms like YouTube, Vimeo, or SoundCloud, we do not upload the file—we embed it.

RTE Feature: The editor must have an "Embed Media" button that accepts a standard URL or a direct iframe code snippet.

Safety Check: On the Node.js backend (PUT /api/content/[id]/save), the application MUST run the resulting HTML through a sanitizer library (like dompurify) to ensure any user-pasted iframe/script tags are from an allow-listed, trusted domain (YouTube, Vimeo, etc.), preventing XSS attacks.
