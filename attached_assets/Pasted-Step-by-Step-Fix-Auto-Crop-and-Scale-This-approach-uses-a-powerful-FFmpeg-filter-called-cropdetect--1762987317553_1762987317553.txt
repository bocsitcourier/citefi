Step-by-Step Fix: Auto-Crop and Scale
This approach uses a powerful FFmpeg filter called cropdetect to automatically find the black bar boundaries and then uses a second pass of filters to remove them and resize the video to a standard 16:9 ratio.

1. Prerequisite: Analyzing the Video (FFprobe)
First, let's use the ffprobe feature within fluent-ffmpeg to get the video's current metadata. You'll need this to ensure the target output is correct.

2. Node.js Code: Dynamic Crop Detection and Re-encode
This code attempts to find the actual dimensions of the video content (excluding the black bars) and then scales that content up to your target resolution (e.g., 1920x1080).

JavaScript

const ffmpeg = require('fluent-ffmpeg');
const path = require('path');

const INPUT_FILE = 'path/to/gemini-output.mp4';
const OUTPUT_FILE = path.join(path.dirname(INPUT_FILE), 'final_no_bars.mp4');

// Target resolution for the final output (16:9)
const TARGET_W = 1920; 
const TARGET_H = 1080;

/**
 * Executes a two-pass FFmpeg command to remove permanent black bars
 * and re-encode the video to a 16:9 aspect ratio.
 */
function fixBlackBars() {
    console.log('Starting Black Bar Removal and Re-encoding...');
    
    // --- Pass 1: Use cropdetect to find the content boundaries ---
    // The output will be a long string containing a "crop=..." command
    ffmpeg(INPUT_FILE)
        .noVideo() // Don't output a video file in this pass
        .outputOptions('-vf', 'cropdetect=limit=0.01:round=16') // Detects crop area
        .on('stderr', (stderrLine) => {
            // FFmpeg prints the crop parameters to stderr
            const cropMatch = stderrLine.match(/crop=(\d+:\d+:\d+:\d+)/);
            if (cropMatch) {
                // Get the 'crop=W:H:X:Y' filter string
                const cropFilter = cropMatch[0]; 
                console.log(`✅ Detected crop filter: ${cropFilter}`);
                
                // --- Pass 2: Apply the detected crop and then scale ---
                reEncodeWithCrop(cropFilter);
                // Stop the first pass once the filter is found
                this.kill('SIGINT'); 
            }
        })
        .on('error', (err) => {
             // We expect this error because we kill the process, ignore it.
             if (!err.message.includes('SIGINT')) {
                 console.error(`❌ Crop detection error: ${err.message}`);
             }
        })
        .run();
}


/**
 * Re-encodes the video using the detected crop filter and scales it to 16:9.
 * @param {string} cropFilter - The 'crop=W:H:X:Y' string from the first pass.
 */
function reEncodeWithCrop(cropFilter) {
    ffmpeg(INPUT_FILE)
        .videoFilter([
            // 1. Crop: Remove the detected black bars (e.g., 'crop=1080:1620:0:150')
            cropFilter,
            
            // 2. Scale: Scale the now cropped content to cover the target area, 
            // sacrificing content to fill the screen (Option 1 logic)
            `scale=${TARGET_W}:-1:force_original_aspect_ratio=increase`,
            
            // 3. Final Crop: Ensure the final output is exactly 1920x1080, centered
            `crop=${TARGET_W}:${TARGET_H}`
        ])
        .outputOptions([
            '-c:v libx264',
            '-crf 23',
            '-c:a aac',
            '-b:a 192k'
        ])
        .on('end', () => {
            console.log(`✅ Success! Video saved to ${OUTPUT_FILE}`);
        })
        .on('error', (err) => {
            console.error(`❌ Re-encode error: ${err.message}`);
        })
        .save(OUTPUT_FILE);
}

fixBlackBars();
How This Works
Pass 1 (cropdetect): The first FFmpeg run doesn't generate a video. It runs the cropdetect filter, which analyzes the video frame-by-frame for areas of pure black. It then prints the necessary crop=W:H:X:Y filter to the console's error stream (stderr).

Filter Extraction: The Node.js code listens to stderr and extracts that exact crop=... string.

Pass 2 (Re-encode): A second FFmpeg process starts.

It applies the crop filter first, which removes the black bars and leaves you with the clean content.

It then applies the standard scale and crop filters from the previous answer (Option 1) to take that clean content and force it to fill the 16:9 canvas.

This dynamic approach is the most reliable way to handle videos where the black bars are already baked into the generated source file.
