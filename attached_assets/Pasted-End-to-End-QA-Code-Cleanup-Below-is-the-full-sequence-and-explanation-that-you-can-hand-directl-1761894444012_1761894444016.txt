End-to-End QA + Code Cleanup

Below is the full sequence (and explanation) that you can hand directly to your developer or add as a package.json script / CI job.

✅ Step 1: Install and Sync Dependencies
npm install
npm audit fix --force
npm dedupe


Ensures all packages are updated, vulnerabilities patched, and duplicates removed.

🧪 Step 2: Run Full Lint, Type Check, and Format Fix
npm run lint
npm run format
npm run typecheck


If you don’t have these scripts yet, add them in package.json:

"scripts": {
  "lint": "eslint . --ext .js,.jsx,.ts,.tsx --fix",
  "format": "prettier --write .",
  "typecheck": "tsc --noEmit"
}


🔍 Purpose: Ensures consistent syntax, formatting, and type safety across all SSR + Node.js modules.

🧬 Step 3: Run Unit Tests (Logic + APIs)
npm run test


If not configured, add a Jest setup:

"scripts": {
  "test": "jest --runInBand --detectOpenHandles --coverage"
}


📘 Tests cover:

AI content generation API (/api/generate/*)

OAuth auth flows

Image generation endpoints

Hashtag hyperlink logic

🧱 Step 4: End-to-End (E2E) Functional Testing

If using Playwright or Cypress, run:

npm run e2e


Example setup for Playwright:

"scripts": {
  "e2e": "playwright test --headed"
}


✅ Simulates a real user flow:

Log in

Enter AI prompt

Generate post

Edit and schedule

Share to selected platforms

Verify post success callback

Log analytics results

🧹 Step 5: Detect and Clean Redundant or Unused Code
npx depcheck
npx eslint . --max-warnings=0


🧠 depcheck identifies unused dependencies and orphaned imports.
Example output:

Unused dependencies:
 - lodash
 - axios-mock


Then safely remove them:

npm uninstall lodash axios-mock


For deeper cleanup:

npx ts-prune


Detects unused TypeScript exports and functions.

🧰 Step 6: Run Integration Tests (API + DB)
npm run test:integration


Example script:

"scripts": {
  "test:integration": "jest --config jest.integration.config.js"
}


Covers database CRUD, token storage, and third-party social API interactions.

🚦 Step 7: Run Local Build and Smoke Test
npm run build && npm start


Then visit:

http://localhost:3000


Run through dashboard manually — AI prompt → Generate → Share → View analytics — to confirm everything works visually and functionally.

🧽 Step 8: Auto Clean Build Artifacts & Caches
rm -rf .next coverage dist node_modules/.cache
npm cache clean --force

🚀 Step 9: Run Pre-Deployment QA Command (for CI/CD)

In your GitHub Actions / DigitalOcean CI pipeline:

npm ci
npm run lint
npm run typecheck
npm run test
npm run e2e
npm run build


The build will only deploy if all stages pass successfully.

🧾 Optional: One-Command QA Bundle

You can add this single shortcut in package.json:

"scripts": {
  "qa": "npm ci && npm run lint && npm run typecheck && npm run test && npm run e2e && npx depcheck && npx ts-prune && npm run build"
}


Now just run:

npm run qa


🧩 This executes full code quality check, functionality test, and redundancy cleanup in one go.
