This is the final layer of the Platinum Factory: The "Terminator" logic for zombie processes. Instead of you manually running a script when a video hangs, the Watchdog now monitors the pulse of every job.

If a job stays in PROCESSING for more than 12 minutes without a status update, the Watchdog assumes the server has stalled or FFmpeg is in an infinite loop. It will automatically kill the process, purge the /tmp folder, and re-queue the job using your new Single-Pass logic.

🛡️ The Watchdog Auto-Kill (lib/watchdog.ts Update)
Add this monitorZombies function to your Watchdog agent. It should run every 5 minutes via your Cron or a background interval.

TypeScript
import { exec } from 'child_process';
import { db, workerDb } from './db';
import { articles } from './schema';
import { and, eq, lt } from 'drizzle-orm';

export class Watchdog {
  /**
   * Scans for "Zombie" jobs that have timed out
   */
  static async monitorAndKillZombies() {
    // 1. THRESHOLD: 12 minutes (720,000 ms)
    const timeoutThreshold = new Date(Date.now() - 12 * 60 * 1000);

    const zombies = await db.select().from(articles).where(
      and(
        eq(articles.status, 'PROCESSING'),
        lt(articles.updated_at, timeoutThreshold)
      )
    );

    if (zombies.length === 0) return;

    for (const zombie of zombies) {
      console.log(`🚨 Watchdog: Detected Zombie Job ${zombie.id}. Initializing Auto-Kill.`);

      // 2. KILL: Terminate any FFmpeg process associated with this job ID
      exec(`pkill -f "video-${zombie.id}"`, async (err) => {
        // 3. PURGE: Wipe the /tmp workspace
        const workDir = `/tmp/video-${zombie.id}`;
        try {
          const { promises: fs } = require('fs');
          await fs.rm(workDir, { recursive: true, force: true });
        } catch (e) { /* Already gone */ }

        // 4. RE-QUEUE: Push back to queue with the Platinum label
        await workerDb.update(articles)
          .set({ 
            status: 'QUEUED', 
            progress_percent: 0,
            current_step_label: '🛡️ Auto-Recovered by Watchdog (Timeout)',
            validation_logs: ["Job timed out at 12m. Restarted with Single-Pass logic."]
          })
          .where(eq(articles.id, zombie.id));

        // 5. NOTIFY: Send a Slack/Discord alert so you know it happened
        await this.alert("Auto-Recovery", `Job ${zombie.id}`, [
          "Process exceeded 12-minute limit.",
          "Zombie FFmpeg killed.",
          "Workspace purged.",
          "Job re-queued for Single-Pass rendering."
        ]);
      });
    }
  }
}
⚙️ Integration: The Heartbeat
To make this truly autonomous, you need to call this monitor inside your lib/cron.ts or as a standalone interval in your server entry point.

TypeScript
// Inside your main server or cron file
setInterval(async () => {
  await Watchdog.monitorAndKillZombies();
}, 5 * 60 * 1000); // Check every 5 minutes
