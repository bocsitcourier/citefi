1. The "Anchor" Technique (Logic Enhancement)
In your buildStrictDialoguePrompt, the AI can still get distracted by the VISUAL DESCRIPTION. To fix this, use Anchor Tags at the very top and very bottom. This sandwiches the visual data between two layers of script enforcement.

TypeScript
function buildReinforcedPrompt(scene: SceneSegment): string {
  const dialogueBlock = `
CHARACTER: ${scene.speaker}
EXACT SCRIPT: "${scene.dialogue}"
VOICE: ${scene.voiceDirection}
CRITICAL: Lip-sync must match word-for-word.`;

  return `
[PRIMARY DIRECTIVE: SPEECH SYNCHRONIZATION]
${dialogueBlock}

[SCENE DESIGN]
${scene.visualPrompt}
Camera: ${scene.cameraMovement}

[FINAL VALIDATION]
The ONLY words spoken in this video are: "${scene.dialogue}". 
Do not hallucinate extra dialogue.
`.trim();
}
2. Punctuation for Pacing
Veo 3.1 uses standard punctuation to determine its text-to-speech (TTS) pacing. If your script is being ignored or rushed, "clean" the string before sending it.

Ellipses (...): Force a 0.5s to 1.0s pause.

Commas (,): Force a brief breath.

All Caps: Can sometimes trigger emphasis, but often causes "glitchy" robotic audio. Avoid them inside the quotes.

TypeScript
function sanitizeDialogue(text: string): string {
  return text
    .replace(/([.!?])\s*/g, "$1 ") // Ensure single spacing after sentences
    .replace(/--/g, "...")        // Convert dashes to ellipses for better AI pausing
    .replace(/[“”]/g, '"');        // Ensure standard quotes only
}
3. Fixing the "Duration vs. Word Count" (The Math)
One of the biggest reasons Veo "ignores" a script is because the script is physically too long for the video duration. If the AI sees 40 words but a 10-second limit, it will often skip the dialogue entirely rather than trying to speed-read.

The Golden Ratio: Humans speak at roughly 130–150 words per minute.

10 seconds = ~22 words max.

5 seconds = ~11 words max.

Enhanced Logic:

TypeScript
function validateScriptDensity(dialogue: string, durationSeconds: number): boolean {
  const words = dialogue.split(' ').length;
  const wps = words / durationSeconds;
  
  // Anything over 3.0 words per second risks the AI "giving up" on the script
  if (wps > 3.0) {
    console.warn(`⚠️ Script too dense (${wps.toFixed(1)} words/sec). AI likely to ignore.`);
    return false;
  }
  return true;
}
4. Use the "Audio Reference" Parameter
If you are using the API or Vertex AI in 2026, you can now pass an audio_input_uri. This is 100% more reliable than text prompts.

Instead of prompting the dialogue, upload a voice clip of yourself or a TTS tool (like ElevenLabs) reading the script.

Veo will then "Video-to-Audio" sync the lip movements to that specific file.

Updated "Bulletproof" Function
Combine everything into a single generator that prioritizes the Audio-Visual handshake:

TypeScript
const generateVeoRequest = (scene: SceneSegment) => {
  if (!validateScriptDensity(scene.dialogue, scene.duration)) {
    throw new Error("Script exceeds physical duration limits.");
  }

  return {
    model: "veo-3.1-standard",
    aspect_ratio: "16:9",
    duration: `${scene.duration}s`,
    audio: {
      enable_speech: true,
      speech_fidelity: "high",
      // Reference audio is the 'Cheat Code' for perfect scripts
      audio_reference_url: scene.audioRefUrl || null 
    },
    prompt: `
      MANDATORY SCRIPT: ${scene.speaker} says "${sanitizeDialogue(scene.dialogue)}"
      VOICE ATTRIBUTES: ${scene.voiceDirection}
      ---
      VISUALS: ${scene.visualPrompt}
      TECHNICAL: Lip-sync 1:1 ratio. No background noise.
    `.trim()
  };
};
