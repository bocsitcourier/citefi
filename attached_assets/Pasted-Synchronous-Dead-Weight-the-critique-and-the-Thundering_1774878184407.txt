The worker will trigger the critique, but it will not wait for the answer. The critique will run in the background and quietly update the database with the score whenever it finishes, while the main pipeline races ahead to Stage 2.Update your worker logic:TypeScriptimport { statelessDb } from "./db";

function launchBackgroundCritique(articleId: string, content: string) {
  console.log(`🕵️‍♂️ Launching background critique for ${articleId}... (Not blocking pipeline)`);
  
  // Notice there is NO 'await' here in the main thread.
  // It runs as an orphaned promise, perfectly safe in Node.js.
  runArticleCritique(content)
    .then(async (score) => {
      // Quietly update the DB whenever it finishes (whether that's in 20s or 6m)
      await statelessDb.update(articles)
        .set({ quality_score: score })
        .where(eq(articles.id, articleId));
      console.log(`📊 Background score saved for ${articleId}: ${score}/100`);
    })
    .catch((err) => {
      console.error(`⚠️ Background critique failed for ${articleId}, ignoring.`, err.message);
    });
}

This spaces out the 10+ parallel jobs so they hit Google's servers like a zipper instead of a brick wall.Add this utility (lib/utils.ts):TypeScript/**
 * Pauses execution for a random amount of time to prevent API throttling.
 * @param minMs Minimum milliseconds to wait
 * @param maxMs Maximum milliseconds to wait
 */
export async function applyJitter(minMs = 5000, maxMs = 25000) {
  const delay = Math.floor(Math.random() * (maxMs - minMs + 1) + minMs);
  console.log(`🚦 Staggering API call: waiting ${(delay / 1000).toFixed(1)}s...`);
  return new Promise((resolve) => setTimeout(resolve, delay));
}
