1. The "Guardian" Verification Loop
This architectural change moves your app from "Fire and Forget" to "Verify and Correct."

2. The Guardian Code (lib/guardian-agent.ts)
This script uses a fast, "sharp" model (like GPT-4o-mini) to act as a cold-blooded auditor. It doesn't write content; it only looks for errors.

TypeScript
import { OpenAI } from "openai";

export interface AuditReport {
  passed: boolean;
  score: number; // 0-100
  missingElements: string[];
  formattingIssues: string[];
  suggestions: string[];
}

export async function auditArticle(html: string, originalRequirements: any): Promise<AuditReport> {
  const openai = new OpenAI();
  
  const prompt = `
    You are an Enterprise Content Auditor. Review the following HTML article against these strict requirements:
    1. Must have at least 2 <img> tags with alt text.
    2. Must have a <h3> FAQ section with at least 3 questions.
    3. Must have at least 5 <a> tags (hyperlinks).
    4. Must follow the "${originalRequirements.persona}" persona tone.
    5. Formatting must be clean (no raw markdown, no broken divs).

    ARTICLE HTML:
    ${html}

    Return ONLY a JSON object with this structure:
    {
      "passed": boolean,
      "score": number,
      "missingElements": string[],
      "formattingIssues": string[],
      "suggestions": string[]
    }
  `;

  const response = await openai.chat.completions.create({
    model: "gpt-4o-mini",
    messages: [{ role: "system", content: prompt }],
    response_format: { type: "json_object" }
  });

  return JSON.parse(response.choices[0].message.content || "{}");
}
3. Integrating the "Self-Correction" Loop
Now, we update your worker.ts so that it doesn't finish until the Guardian is happy.

// Inside your generation worker...
let attempts = 0;
let isArticleReady = false;
let finalHtml = "";

while (attempts < 2 && !isArticleReady) {
  // Step 1: Generate or Reformat
  finalHtml = await generateArticleContent(batch, facility);

  // Step 2: Run the Guardian Audit
  const audit = await auditArticle(finalHtml, { persona: batch.persona });

    isArticleReady = true;
    console.log("✅ Guardian approved article.");
  } else {
    attempts++;
    console.warn(`⚠️ Guardian rejected article (Attempt ${attempts}). Issues:`, audit.missingElements);
    
    // Step 3: Surgical Fix (Pass the audit suggestions back to GPT to fix ONLY the gaps)
    finalHtml = await surgicalFix(finalHtml, audit.suggestions);
  }
}

4. Closing the "Intelligence" Gap (The Style Ledger)
To make the app "get smarter," we need to store the Guardian's failures. If the Guardian constantly finds "Missing FAQs," the system should learn to emphasize FAQs in the next primary prompt.

The Feature: Create a table called generation_feedback.

The Logic: Every time the Guardian finds a "Missing Element," log it.

The Optimization: Before the next article is written, the app runs a quick query: "What are the top 3 issues we've had this week?" If "Missing Images" is #1, it injects a high-priority warning into the Master Prompt: "CRITICAL: You have been failing to include images. Do not skip the <img> tags this time."

🏗️ Why this is "Platinum" level:
Deterministic Quality: You stop getting "lucky" with good articles. The code guarantees them.

Proactive Suggestions: Since the Guardian returns a suggestions array, you can display a "Pro-Tip" box next to the article in the UI: "The AI noticed your local SEO keyword density is low—click 'Optimize' to boost it."

Unified Memory: By using the generation_feedback loop, the app actually "remembers" its own mistakes and corrects its behavior over time.
