Prompt design — explicit, constrained instructions

🧱 Core Solution Structure
Stage	Goal	Implementation
1. Input enforcement	Never let a request run without a valid businessName.	Validate JSON schema or API payload.
2. Prompt injection	Insert the exact name verbatim and early in every prompt.	Use a standardized template across modules.
3. Validation layer	Confirm name integrity in every output.	Regex for text, OCR for images.
4. Fallback	Correct any mismatch automatically.	Replace or overlay correct name.
🧠 1. Article Generation Prompt Template

Goal: Ensure the model never rephrases or mis-spells the business name.

Prompt structure:

System / Instruction layer:
You are generating marketing and SEO content for a business. 
Always use the company name exactly as provided. 
Never invent, modify, abbreviate, or translate it. 
The name must appear verbatim as: "{businessName}"

User prompt:
Write a 1500-word article about {topic}. 
The article must include the company name "{businessName}" exactly as shown throughout. 
Do not use synonyms or variations. 
Emphasize the company’s professionalism and reliability. 


Example:

The exact company name is "PRIVATE INHOME CAREGIVER". 
Use this exact text in the article title and throughout the content without any change.


Post-validation:
After generation → run a simple regex:

if businessName not in generated_text:
    regenerate_or_replace_name(generated_text, businessName)

🎨 2. Image Generation Prompt Template

Goal: Ensure images show only the correct name, or stay blank for overlay.

Option A — Render exact text

Create a professional hero image for the company. 
The image must include the exact text "{businessName}" in a large, clear font. 
Do not invent, change, or misspell the name. 
If you cannot render the exact text, leave the space blank.


Option B — Placeholder for programmatic overlay (preferred for reliability)

Create a professional hero image for a home care website. 
Leave an empty sign or banner space at the top center for text overlay. 
Do not include any text. 
The text will be added later as "{businessName}".


OCR verification (for Option A):

ocr_text = run_ocr(image)
if businessName not in ocr_text:
    overlay_text(image, businessName)

🔄 3. Centralized Prompt Composer (Best Practice)

All modules (article, image, audio, video) should call a single function:

def build_prompt(template, businessName, topic):
    validated_name = sanitize(businessName)
    if not validated_name:
        raise ValueError("Missing or invalid business name")
    return template.replace("{businessName}", validated_name)


This guarantees consistent prompt injection and eliminates per-module drift.

🧰 4. Brand Validator Middleware

A lightweight service that runs after generation:

Content Type	Validator	Correction
Text	Regex exact match for businessName	Replace or regenerate
Image	OCR + string match	Overlay correct text
Audio/Video	Speech-to-text match	Re-dub name via TTS overlay
🚀 Combined Workflow Overview

Receive request → includes topic, businessName, mediaTypes

Validate input → reject or fix missing name

Compose prompts via build_prompt() for all modules

Generate content (article, image, audio, etc.)

Run brand validator

If mismatch → auto-correct (overlay or regenerate)

Publish / Store → mark verified ✅
