ApexContent Engine – Full Developer Handoff Document (SSR, Node.js, DigitalOcean Deployment)
1. System Overview

Purpose:
A high-speed, scalable AI content generation web app that produces SEO- and GEO-optimized articles with social media posts. Supports batch article generation, rich editing, media uploads, internal linking, semantic clusters, and localized SERP optimization.

Stack:

Frontend: Next.js 14+ with SSR (App Router)

Backend: Node.js 20+ with TypeScript

Database: PostgreSQL (Managed via DigitalOcean)

Job Queue / Async Processing: BullMQ (Redis-backed)

Object Storage: DigitalOcean Spaces (S3-compatible)

AI Services: Gemini API, GPT-4 API

Social API Integration: X (Twitter), Facebook, Instagram, Pinterest, LinkedIn

Deployment: DigitalOcean App Platform or Droplets, Docker optional

4. API Specifications
Job & Article Management
Endpoint	Method	Purpose	Input Body/Params	Output
/api/jobs/title-pool	POST	Generate 50-title pool	core_topic: string, numTitles: int	pool: string[], primarySEO: object
/api/jobs/batch-submit	POST	Submit bulk job	userId: int, batchConfig: object	batchId, status

/api/articles/[id]	GET	Retrieve full article	article_id	article object (HTML, SEO data, assets)
/api/articles/[id]	PUT	Update article	updates JSON (slug, meta, keywords, hashtags, internal links)	updated article object
/api/social_posts/generate	POST	Generate AI social posts	article_id, platforms[], tone, mood, industry	posts[]
/api/social_posts/schedule	POST	Schedule auto-share	social_post_id, schedule_at	status
AI Processing Workflow (Async Jobs)

Batch Queue (BullMQ + Redis)

Jobs: article generation, image processing, GPT-4 SEO/Geo optimization, social post generation.

Retry logic & concurrency limits.

Stage 1 – Gemini Draft Generation

Input: user topic, competitor URLs, target geo, semantic cluster, desired SERP feature.

Output: structured JSON (article_text, subheadings, image prompts, keywords, hashtags).

Stage 2 – Asset Generation

Input: image/video prompts from Stage 1.

Processing: Sharp / FFmpeg to optimize and convert images/videos to web-friendly formats.

Output: Storage URLs for HTML embedding.

Stage 3 – GPT-4 SEO & Geo Optimization

Input: draft article, internal link suggestions, competitor data.

Actions: Insert hyperlinks, format H1/H2/H3, validate schema markup (Article, FAQ, HowTo).

Output: final_html_content, meta, slug, geo score, suggested social posts.

Stage 4 – Social Post Generation

Input: final_html_content, tone/mood, platform, industry.

Output: 3–5 smart posts per platform with hashtags, emojis, AI-generated image/video.

6. Deployment (DigitalOcean)

App Platform or Droplets: Node.js with Next.js SSR.

Environment Variables: Gemini & GPT API keys, DB credentials, DO Spaces credentials.

Auto-Scaling: At least 2 instances for high concurrency.

PM2: Process management and auto-restart for job workers.

Redis: Hosted or self-managed for BullMQ queue.

Backups: Daily PostgreSQL snapshot & Spaces backup.

7. SEO & Geo Optimization Rules

Inverted pyramid style with BLUF first.

H2/H3 question-based headings, bulleted lists, tables, FAQs.

Semantic keywords, internal linking, structured schema markup.

Geo-accuracy score 0–100; local entity inclusion mandatory.

Competitor URLs used as grounding reference for content coverage.

SERP targeting (Featured Snippet, PAA, Lists) adjusts prompt and formatting.

Cost-per-rank tracking and content refresh suggestions included.
