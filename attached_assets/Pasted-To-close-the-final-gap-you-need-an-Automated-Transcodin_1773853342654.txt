Fix 7: Automated Transcoding Pipeline (ABR Strategy)
The Problem: Raw AI video is "one-size-fits-all." A 1080p 30Mbps clip is overkill for a mobile screen and impossible for slow networks.
The Solution: Implement Adaptive Bitrate Streaming (ABR). This creates multiple versions of every video (360p, 720p, 1080p) and swaps them in real-time based on the user's internet speed.

Technical Requirements for the Architect:
Protocol: Use CMAF (Common Media Application Format). This is the gold standard for 2026. It allows you to store one set of video fragments that work for both HLS (Apple) and DASH (Android/Web), cutting your storage costs in half.

Trigger: Set up a Cloud Pub/Sub or S3/GCS Event trigger. The moment a Veo video is saved to storage, it should automatically spin up a Transcoder API job.

Job Templates: Create three "Rendition Profiles":

Mobile High (720p): Optimized for small screens and efficiency.

Desktop High (1080p): Full fidelity for high-end users.

Data Saver (360p): For low-bandwidth environments.

FastStart Optimization: If you continue using MP4s for social sharing, use the -movflags faststart flag in FFmpeg to move metadata to the front of the file so playback starts instantly.
