🛠️ Detailed FFmpeg Instructions (Node.js)
You should have fluent-ffmpeg installed (npm install fluent-ffmpeg).

Option 1: Crop to Fill (Removes Black Bars) 🎯
This is the standard fix. It removes the black bars by scaling the video up until the sides touch the edges of the 16:9 container, and then cropping off the excess video data from the top and bottom (or vice-versa, depending on aspect ratios). This sacrifices some content but fills the screen.

Code for 16:9 Output (e.g., 1920x1080)
JavaScript

const ffmpeg = require('fluent-ffmpeg');
const INPUT_FILE = 'path/to/gemini-output.mp4';
const OUTPUT_FILE = 'path/to/final-16-9-cropped.mp4';

// Desired final resolution (e.g., 16:9 standard HD)
const TARGET_W = 1920; 
const TARGET_H = 1080;

ffmpeg(INPUT_FILE)
    .videoFilter([
        // 1. Scale: Force the video to scale up until its *shortest* side (the width)
        // covers the target width (1920), making the height much larger than 1080.
        // This is key for vertical video to fill horizontal space.
        `scale=${TARGET_W}:-1:force_original_aspect_ratio=increase`,
        
        // 2. Crop: Cut the scaled video to the exact desired height (1080), 
        // centering the crop vertically.
        `crop=${TARGET_W}:${TARGET_H}`
    ])
    .outputOptions([
        '-c:v libx264',
        '-crf 23',
        '-c:a aac',
        '-b:a 192k'
    ])
    .on('end', () => {
        console.log(`✅ Success! Video processed and saved to ${OUTPUT_FILE}`);
    })
    .on('error', (err) => {
        console.error(`❌ FFmpeg Error: ${err.message}`);
    })
    .save(OUTPUT_FILE);

Note: The -1 in the scale filter tells FFmpeg to automatically calculate the height while maintaining the aspect ratio, which is crucial before the cropping step.

Option 2: Aspect Ratio Change (Squashing/Stretching) 📐
If you cannot afford to lose any content from the top/bottom, the only other way to remove the bars and fill the screen is to stretch the video horizontally. This is generally discouraged as it distorts the image.

Code for Stretching to 16:9
JavaScript

const ffmpeg = require('fluent-ffmpeg');
const INPUT_FILE = 'path/to/gemini-output.mp4';
const OUTPUT_FILE = 'path/to/final-16-9-stretched.mp4';

// Desired final resolution
const TARGET_W = 1920; 
const TARGET_H = 1080;

ffmpeg(INPUT_FILE)
    .videoFilter([
        // Force the video to be exactly 1920x1080, ignoring its original aspect ratio.
        `scale=${TARGET_W}:${TARGET_H}` 
    ])
    .outputOptions([
        '-c:v libx264',
        '-crf 23',
        '-c:a aac',
        '-b:a 192k'
    ])
    .on('end', () => {
        console.log(`✅ Success! Video processed and saved to ${OUTPUT_FILE}`);
    })
    .on('error', (err) => {
        console.error(`❌ FFmpeg Error: ${err.message}`);
    })
    .save(OUTPUT_FILE);
❓ If the Issue Persists
If you run Option 1 and still see black bars, it means the video output from the Gemini API might have an already padded frame. The actual content might be smaller than the video file's overall dimensions.

To debug this, you'll need to use FFmpeg's analysis tools to confirm the source video's dimensions and frame size:

Run this command in your terminal (not Node.js) on the source file:

Bash

ffprobe -v error -select_streams v:0 -show_entries stream=width,height -of csv=p=0 input_file.mp4
This will output the Width,Height (e.g., 1080,1920). If the height is greater than the width, it's a vertical video and Option 1 is correct.

If the aspect ratio is very strange, you may need to add a setsar=1 (Set Sample Aspect Ratio) filter to ensure square pixels before applying the crop/scale, which can sometimes fix unusual encoding issues.
