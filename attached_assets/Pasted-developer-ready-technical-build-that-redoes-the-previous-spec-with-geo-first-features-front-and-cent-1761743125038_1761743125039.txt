ApexContent Engine — Geo-First Technical Build & Developer Schema
Executive Summary

ApexContent Engine is an AI-driven platform that generates up to 50 SEO-optimized, geo-targeted articles per batch, including local metadata, regionally-aware content, social blurbs (emojis + hashtags), and one-click export/publishing to CMS (WordPress), Google Sheets, XML sitemaps, and Markdown. This build emphasizes geographic relevance — automatically injecting local keywords, coordinates, region-specific schema, localized hashtags, and regional variants of keywords to maximize local search performance.

1. Customer Journey (Geo-first)

Onboard / Setup

User creates account, sets default locale(s), brand voice, and business NAP (Name, Address, Phone).

Optional: connect Google Business Profile / GSC for future analytics.

Project Setup (Wizard)

Step 1: Topic + Primary Keyword

Step 2: Choose Primary Locale(s) — Country > State/Province > City > ZIP/Postal (supports multiple locales per batch)

Step 3: Brand Voice + Tone

Step 4: Article Count (1–50) + Word Count Range

Step 5: Local SEO Options (auto add LocalBusiness schema, insert NAP, include local landmarks, hours, phone)

Generation

User triggers batch generation → queued job(s) spawn per article.

Each article runs triage pipeline: Outline → Draft → Geo-Enrich → SEO Finalize → Asset generation → Post-blurb generation.

Review & Edit

Editor shows final HTML with geo meta tags, JSON-LD LocalBusiness, image assets, and SEO score.

User can edit and preview SERP / social cards.

Publish / Export

Options: WordPress publish (REST), Google Sheets push, CSV/ZIP download, XML sitemap generation, Markdown export.

Optionally schedule or webhook to third-party publishers.

(Optional) Monitor

Future: connect to Google Search Console / GA4 for ranking / traffic analytics and automated re-optimization suggestions.

5. AI Pipeline — Geo-aware Prompts & Structured Outputs

Each article goes through 3 main AI steps — made geo-aware with added context:

5.1: generateArticleCore(title, config) — Gemini

System instruction (template):

You are an expert local SEO content writer. Create a unique, {wordCount}-word article for the title "{title}" targeted to {city}, {region}, {country}. The content must:
- Use US/UK spelling depending on language.
- Mention local landmarks, neighborhoods, or municipalities when appropriate.
- Include NAP where relevant if provided.
- Provide 5 local long-tail keywords and 10-15 geo-specific hashtags.
Return a single JSON object strictly matching the following schema (application/json).


Schema

{
  "type":"object",
  "properties":{
    "article_text":{"type":"string"},
    "slug":{"type":"string"},
    "meta_description":{"type":"string"},
    "long_phrase_keywords":{"type":"array","items":{"type":"string"}},
    "hashtags":{"type":"array","items":{"type":"string"}},
    "image_prompts":{"type":"array","items":{"type":"string"}},
    "local_references":{"type":"array","items":{"type":"string"}} -- e.g., "Back Bay, Logan Airport"
  },
  "required":["article_text","slug","meta_description","long_phrase_keywords","image_prompts"]
}


Key prompt injectables: $title, $wordCount, $city, $region, $country, $nap, $language, $audience.

5.2: generateAssetsAndStore(imagePrompts, slug)

For each image_prompt, call image API with locale tags (e.g., “photorealistic photo of Boston skyline, morning, brownstone houses, include local license plate MA”).

Convert images to WebP via sharp.

Upload to Spaces/S3 using naming convention /{batch_id}/{slug}-{index}.webp.

Return storage_url and alt_text (alt_text should mention location — e.g., “Boston logistics courier van on Cambridge street”).

5.3: finalizeContent(articleData, targetUrl) — GPT-4

Input: article_text, keywords, hashtags, storage_url array, local_references, NAP.

Tasks:

Inject HTML structure (H1, H2, H3).

Insert 3–5 internal/external links; primary CTA (targetUrl).

Add LocalBusiness JSON-LD if includeLocalBusiness is true.

Place images with alt text and loading="lazy".

Build hreflang tags if translations exist.

Return final_html_content and seo_score (calculated via script).

Example LocalBusiness JSON-LD (to inject into <head> and final_html_content):

{
  "@context": "https://schema.org",
  "@type": "LocalBusiness",
  "name": "Acme Courier",
  "image": "https://cdn.example.com/batch/slug-1.webp",
  "@id": "https://example.com/#acme-courier",
  "url": "https://example.com",
  "telephone": "+15555555555",
  "address": {
    "@type": "PostalAddress",
    "streetAddress": "122 Main St",
    "addressLocality": "Boston",
    "addressRegion": "MA",
    "postalCode": "02108",
    "addressCountry": "US"
  },
  "geo": {
    "@type": "GeoCoordinates",
    "latitude": 42.3601,
    "longitude": -71.0589
  },
  "openingHoursSpecification": [{
     "@type": "OpeningHoursSpecification",
     "dayOfWeek": ["Monday","Tuesday","Wednesday","Thursday","Friday"],
     "opens":"09:00",
     "closes":"18:00"
  }]
}

6. Geo SEO Technical Details
6.1 Slug Strategy

Use slug pattern that includes local specificity when requested:

same-day-courier-boston-ma or same-day-courier-boston-ma-providence-ri for multi-target.

Maintain slug uniqueness with suffixes (e.g., -1, -2) if collisions occur.

Store canonical_url in articles table.

6.2 Hreflang

If multiple languages/regions exist for same content, produce hreflang_json:

[
  {"lang":"en-US","url":"https://example.com/en-us/same-day-courier-boston"},
  {"lang":"es-US","url":"https://example.com/es-us/courier-boston"}
]


Inject <link rel="alternate" hreflang="en-US" href="..."> in head.

6.3 Geo Meta Tags (for enhanced local relevance)

Insert:

<meta name="geo.region" content="US-MA">
<meta name="geo.placename" content="Boston">
<meta name="geo.position" content="42.3601;-71.0589">
<meta name="ICBM" content="42.3601,-71.0589">

6.4 Local Keyword Variants

Generate variants: [keyword] + city, [keyword] + neighborhood, [keyword] + zip. E.g., same day courier Boston, same day courier Back Bay, same day courier 02108.

Store these in local_keywords_json.

6.5 NAP & Citation Consistency

If includeLocalBusiness is true:

Insert NAP on article bottom and in schema.

Provide citation_suggestions from Geo Service (e.g., local directories).

6.6 Google My Business / Maps

If user links GMB account (optional), push auto-created article summaries as posts (if API allows).

Use GMB details to cross-verify NAP.

8. Queue & Worker Considerations (Geo heavy)

Use BullMQ + Redis with concurrency limits per worker to avoid rate-limits on AI APIs.

Each article job includes locale metadata.

Concurrency strategy:

maxConcurrentArticles = min(5, floor(API_quota/estimatedTokensPerArticle))

Retry/backoff strategy: exponential backoff with jitter for rate-limit (429).

Worker logging: include locale, city, batchId, articleId, tokensUsed, costEstimate.

10. SEO Scoring & QA

SEO Analyzer (post-finalize) computes:

Title length (<= 60 char)

Meta length (<= 160)

H1 present

Local keywords coverage (at least X usages across body)

Image alt text presence (geo-aware alt text)

LocalBusiness Schema correctness (JSON-LD validators)

Readability score (Flesch)

Return seo_score 0–100 and issues list for editor.

11. Geocoding & Local Data Sources

Integrations:

Google Places API: Place IDs, address components, lat/lng, neighborhood suggestions.

Mapbox (backup/alternative).

GeoNames for city/state robust lists.

Optional: Local keyword APIs (Semrush/Ahrefs) or SerpApi for competitor scraping.

Caching:

Cache place lookups and commonly requested locales in Redis to reduce API hits and latency.

12. Security & Privacy

Never expose API keys or credentials to client.

Use server-side proxy for Geocoding calls (to hide key).

Store PII (NAP) encrypted at rest.

RBAC: teams, agency roles, admin roles. Limit publishing actions to authorized users.

Rate-limit user requests (both UI and API) to control cost.

13. Monitoring, Observability, Cost Controls

Metrics to track:

Queue length per region

Average time per article generation

Token usage per region

Failed jobs per error type

Alerts:

If token/billing exceeds threshold

Spike in 429 errors from external APIs

Logs:

Structured logs include batchId, locale, articleId, tokensUsed.

Dashboard: Admin UI to view cost per batch and ability to pause new jobs.

14. Testing Strategy

Unit tests for prompt generation and response schema validation.

Integration tests that mock AI APIs (return canned JSON) to validate pipeline.

End-to-end test: create batch → process 2 articles → validate final HTML contains geo schema and local keywords.

Load tests: simulate 50-article batch with concurrency to ensure DB + worker stability.
