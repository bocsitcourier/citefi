Why It’s Happening

No canonical variable lock:
Each module (article, image, audio) is referencing the company name from free-text prompts, not from a single variable.
→ Example: “Generate an article for Bocsite Courier” vs “Generate image for Bocsit Delivery Service” → inconsistent spellings.

Autocorrect / LLM bias:
Gemini or GPT models often “normalize” rare or brand-like names into dictionary words unless explicitly told not to correct spelling.

No confirmation step in UI:
The generator doesn’t pause to confirm the name or display it before it’s pushed to the generation queue.

✅ Fix Plan (Full Stack)
1. Centralize Brand Data

Create a global config (e.g. /app/config/branding.ts):

export const BRAND = {
  name: "Bocsit",
  tagline: "Same Day Courier & Delivery Services",
  website: "https://bocsit.com",
  spellcheck_lock: true
};


Then everywhere you use the brand name — article, image, TTS — you import:

import { BRAND } from "@/config/branding";


And in prompts:

`Write an SEO-optimized article about ${BRAND.name}. Do NOT alter the spelling of "${BRAND.name}".`

2. Add a Frontend Confirmation Step

Before generation, show a small modal:

Confirm Company Name
Please confirm the spelling and name before generation:
[ Bocsit ]
✅ Confirm and Generate | ✏️ Edit

Code snippet:

const [companyName, setCompanyName] = useState(BRAND.name);
const [confirmed, setConfirmed] = useState(false);

const handleGenerate = () => {
  if (!confirmed) {
    openConfirmModal(companyName);
    return;
  }
  startGeneration({ companyName });
};


This ensures human confirmation before any job hits the backend.

3. Lock Brand Name in Prompts

Example — Article generation:

const prompt = `
Generate an engaging article for ${companyName}.
IMPORTANT: Do not correct or modify the spelling of the company name "${companyName}" under any circumstances.
Use it exactly as written.
`;


For image generation:

const imagePrompt = `Create a professional image for ${companyName}. 
Do NOT change the spelling "${companyName}". 
The logo and text should exactly reflect "${companyName}".`;


For audio (Gemini TTS):

const audioPrompt = `Narrate this article clearly, pronouncing the brand "${companyName}" exactly as spelled: "${companyName}".`;

4. Enforce Brand Consistency in Output

After generation, run a validation check:

if (!output.includes(companyName)) {
  throw new Error(`Brand name mismatch in output. Expected "${companyName}"`);
}


This blocks saving any content that uses an incorrect spelling.

5. (Optional) Add Voice Pronunciation Mapping

If Gemini TTS pronounces the name wrong:

Use SSML to define pronunciation:

<speak>
  Welcome to <sub alias="Bok-sit">Bocsit</sub> Courier Services.
</speak>


That ensures the brand sounds correct in audio output.
