# Migration Guide: Fixing OpenAI Timeout & Concurrency Issues

## 🔴 Problems Identified

### Current Issues:
1. **Timeout too long**: 180 seconds (3 minutes) - Jobs hang forever
2. **Too many concurrent requests**: 35 simultaneous requests
3. **Worker/slot mismatch**: 50 workers competing for 35 slots
4. **No queue management**: Requests pile up causing rate limit backlog
5. **No circuit breaker**: System keeps retrying failures indefinitely

### Impact:
- Requests timeout after 3 minutes of waiting
- Rate limits hit frequently
- Queue backlog grows exponentially
- Poor user experience (long waits)
- Wasted API costs on timeouts

## ✅ Solutions Implemented

### 1. Reduced Timeouts
```javascript
// BEFORE
timeout: 180000 // 3 minutes 😱

// AFTER
timeout: 30000 // 30 seconds ✅
timeouts: {
  'gpt-4-turbo': 45000,  // 45s for complex
  'gpt-4': 60000,        // 60s for full GPT-4
  'gpt-3.5-turbo': 20000 // 20s for fast model
}
```

### 2. Strict Concurrency Control
```javascript
// BEFORE
maxConcurrent: 35 // Too many! 😱

// AFTER
maxConcurrent: 15 // Optimized limit ✅
interval: 1000    // 1 second between batches
intervalCap: 10   // Max 10 requests per second
                  // = 600 requests/minute throughput
```

### 3. Worker Pool Alignment
```javascript
// BEFORE
workers: 50 // Competing for 35 slots 😱

// AFTER
workers: 15 // Matches concurrency limit ✅
taskTimeout: 45000 // 45s per task
```

### 4. Smart Queue with Priorities
```javascript
// NEW
priorities: {
  critical: 10,  // User-facing requests
  high: 7,       // Important content
  normal: 5,     // Standard generation
  low: 3,        // Background tasks

```

### 5. Circuit Breaker Pattern
```javascript

  failureThreshold: 5,  // Open after 5 failures
  resetTimeout: 60000,  // Reset after 1 minute
  monitorInterval: 5000 // Check every 5 seconds

```

## 🔧 Step-by-Step Migration

### Step 1: Install Dependencies

```bash
npm install p-queue p-retry
```

### Step 2: Update Your AEO Content Factory

Replace the old OpenAI initialization:

```javascript
// OLD CODE (Remove this)
const OpenAI = require('openai');
this.gpt4 = new OpenAI({ 
  apiKey: config.openaiApiKey 
});
```

```javascript
// NEW CODE (Add this)
const { SmartOpenAIClient } = require('./optimized-openai-config');
this.gpt4 = new SmartOpenAIClient({ 
  apiKey: config.openaiApiKey 
});
```

### Step 3: Update API Calls

Replace direct OpenAI calls with the smart client:

```javascript
// OLD CODE (Remove)
const response = await this.gpt4.chat.completions.create({
  model: 'gpt-4-turbo-preview',
  messages: [{ role: 'user', content: prompt }],
  temperature: 0.7,
  max_tokens: 2000
});
return response.choices[0].message.content;
```

```javascript
// NEW CODE (Replace with)
const response = await this.gpt4.chat(
  [{ role: 'user', content: prompt }],
  {
    model: 'gpt-4-turbo',
    temperature: 0.7,
    maxTokens: 2000,
    priority: 'high' // or 'normal', 'low', etc.
  }
);
return response.choices[0].message.content;
```

### Step 4: Update Batch Processing

Replace Promise.all with smart batch processing:

```javascript
// OLD CODE (Remove) - No concurrency control
const [social, podcast, video] = await Promise.all([
  this.generateSocialMedia(article),
  this.generatePodcastScript(article),
  this.generateVideoScript(article)
]);
```

```javascript
// NEW CODE (Replace with)
const tasks = [
  {
    messages: [{ role: 'user', content: socialPrompt }],
    options: { maxTokens: 600, priority: 'high' }
  },
  {
    messages: [{ role: 'user', content: podcastPrompt }],
    options: { maxTokens: 4000, priority: 'normal' }
  },
  {
    messages: [{ role: 'user', content: videoPrompt }],
    options: { maxTokens: 4000, priority: 'normal' }
  }
];

const results = await this.gpt4.batchProcess(tasks, {
  batchSize: 2 // Process 2 at a time
});

const [social, podcast, video] = results.map(r => 
  r.status === 'fulfilled' ? r.value.choices[0].message.content : null
);
```

### Step 5: Add Error Handling

Implement proper timeout and rate limit handling:

```javascript
// NEW CODE - Add this wrapper
async function safeGenerate(client, messages, options = {}) {
  try {
    const response = await client.chat(messages, options);
    return response.choices[0].message.content;
    
  } catch (error) {
    if (error.name === 'TimeoutError') {
      console.error(`Timeout after ${options.timeout || 30000}ms`);
      // Fallback to simpler/faster model
      return await client.chat(messages, {
        ...options,
        model: 'gpt-3.5-turbo',
        maxTokens: Math.min(options.maxTokens || 2000, 1000)
      });
    }
    
    if (error.status === 429) {
      console.error('Rate limit hit - waiting before retry');
      await new Promise(resolve => setTimeout(resolve, 5000));
      return await client.chat(messages, options);
    }
    
    throw error;
  }
}
```

#### High Volume (> 500 requests/hour)
```javascript
// Consider using multiple API keys with load balancing
const clients = [
  new SmartOpenAIClient({ apiKey: process.env.OPENAI_API_KEY_1 }),
  new SmartOpenAIClient({ apiKey: process.env.OPENAI_API_KEY_2 }),
  new SmartOpenAIClient({ apiKey: process.env.OPENAI_API_KEY_3 })

// Round-robin load balancer
let currentClient = 0;
function getClient() {
  currentClient = (currentClient + 1) % clients.length;
  return clients[currentClient];
}
```

## ⚠️ Common Issues & Solutions

### Issue 1: "Queue taking too long"
**Solution:** Increase concurrency slightly (but not above 10)
```javascript
client.queue.concurrency = 8;
client.queue.intervalCap = 5;
```

### Issue 2: "Still getting timeouts"
**Solution:** Reduce maxTokens or use faster model
```javascript
// Reduce token limit
options.maxTokens = Math.min(options.maxTokens, 1500);

// Or fallback to faster model
options.model = 'gpt-3.5-turbo';
```

### Issue 3: "Circuit breaker opening too often"
**Solution:** Increase failure threshold
```javascript
CONFIG.openai.circuitBreaker.failureThreshold = 10; // Up from 5
```

### Issue 4: "Need faster processing for critical tasks"
**Solution:** Use priority system
```javascript
// Critical user-facing requests
await client.chat(messages, { priority: 10 });

// Background batch jobs
await client.chat(messages, { priority: 1 });
```

## 🎯 Best Practices

### 1. Always Set Appropriate Timeout
```javascript
// Short responses (social media)
timeout: 20000 // 20s

// Medium responses (articles)
timeout: 30000 // 30s

// Long responses (detailed content)
timeout: 45000 // 45s

// Complex analysis
timeout: 60000 // 60s MAX
```

### 2. Use Priority System
```javascript
// User-facing = high priority
await client.chat(messages, { priority: 'high' });

// Batch generation = low priority
await client.batchProcess(tasks, { priority: 'batch' });
```

### 3. Monitor Metrics
```javascript
// Check metrics every 100 requests
if (client.metrics.totalRequests % 100 === 0) {
  const metrics = client.getMetrics();
  console.log('Success rate:', (metrics.successfulRequests / metrics.totalRequests * 100).toFixed(1) + '%');
  console.log('Avg response time:', Math.round(metrics.averageResponseTime) + 'ms');
}
```

### 4. Handle Failures Gracefully
```javascript
async function generateWithFallback(client, messages, options) {
  try {
    // Try primary model
    return await client.chat(messages, options);
  } catch (error) {
    console.warn('Primary failed, trying fallback...');
    
    // Fallback to faster/simpler model
    return await client.chat(messages, {
      ...options,
      model: 'gpt-3.5-turbo',
      maxTokens: Math.min(options.maxTokens, 1000)
    });
  }
}
```

### 5. Batch Efficiently
```javascript
// Process in small batches with delays
for (let i = 0; i < tasks.length; i += 3) {
  const batch = tasks.slice(i, i + 3);
  await client.batchProcess(batch);
  
  // Small delay between batches
  if (i + 3 < tasks.length) {
    await new Promise(resolve => setTimeout(resolve, 2000));
  }
}
```
