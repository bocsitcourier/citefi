1. System Architecture & Deployment

Architecture Overview:

User Browser
    ↓
Next.js SSR Server (Pages + API Routes)
    ↓
Job Queue Worker (BullMQ / Redis)
    ↓
AI Services:
   - Gemini API → Draft Article + Hero Image
   - GPT-4 API → Optimization, Hyperlinking, SEO & Schema
    ↓
PostgreSQL → Stores Users, Jobs, Articles, Assets, SEO Logs
DigitalOcean Spaces → Stores Optimized Images/Media


Deployment Notes:

Next.js app is server-side rendered (SSR) for SEO.

Use DigitalOcean App Platform for auto-scaling, high CPU/memory tiers for fast AI generation.

Environment variables for API keys, DB credentials, and Spaces credentials.

3. Async Job Queue & AI Workflow

Queue: BullMQ on Redis

Steps:

User submits batch → /api/jobs/batch-submit → job stored in DB + queued

Title Pool Generation → Gemini API → generate 50 titles

Competitor Grounding → summarize top 3–5 competitor subheadings/entities

Geo-SEO Draft → Gemini generates article draft + hero image

GPT-4 Optimization → hyperlink keywords, insert schema markup, semantic cluster links, SERP feature formatting

Technical SEO Validation → check headings, schema, HTML

Geo-Accuracy Scoring → calculate 0–100 score

Save Article & Assets → DB + DigitalOcean Spaces

Admin Dashboard Updates → live progress, token cost, geo/SEO score

4. API Specifications

Job Batch

POST /api/jobs/batch-submit
Body: {
  user_id,
  core_topic,
  num_articles,
  target_url,
  competitor_urls,
  topic_cluster,
  serp_feature
}
Response: { batch_id, status }

GET /api/jobs/status/[id]
Response: {
  batch: { id, status, articles_generated, seo_score, geo_accuracy_score }
}


Article CRUD

GET /api/articles/[id]
PUT /api/articles/[id]
PATCH /api/articles/[id]/refresh
DELETE /api/articles/[id]
Body (PUT/PATCH): {
  final_html_content,
  slug,
  seo_title,
  meta_description,
  keywords_json,
  hashtags_json,
  internal_link_suggestions
}


Admin Reporting

GET /api/admin/seo-report
Response: {
  article_id,
  tokens_used,
  cost_estimate,
  rank_before,
  rank_after,
  cost_per_rank
}

Geo-Optimization: Local landmarks, entities, slang, seasonal trends

Content Refresh Mode: Update content based on rank drop
