🏗️ The Architect's Fix: The "Guardian Agent" Pipeline
You shouldn't trust GPT-4 to follow a 2,000-word instruction list perfectly every time. Even the best models "hallucinate" or skip steps. You need a Verification Layer (a Checklist Agent) that acts as a quality-control gate.

1. Unified Prompting (The Single Source of Truth)
We need to move your instructions out of the code and into a MasterConfiguration object.

The Gap: Right now, generateArticle.ts has the "Good Instructions," but reformatRoute.ts has a "Short Version."

The Fix: Create a lib/prompts/master-instructions.ts. Every single route—Generate, Reformat, and Regenerate—must pull from this same object. If you change a rule for an article, it updates across the entire app instantly.

2. The "Pre-Flight" Checklist (The Guardian)
Before an article is marked "Complete," a separate, lightweight AI call (The Guardian) should run a checklist against the HTML.

The Guardian's Checklist:

[ ] Does the HTML contain at least 2 <img> tags?

[ ] Is there an <h3>FAQ</h3> section at the end?

[ ] Are there at least 5 internal hyperlinks?

[ ] Does the tone match the "Hungry Disruptor" (or selected Persona)?

If any box is [ ], the Guardian sends it back for "Auto-Correction" before the user ever sees it.

🧠 Why the App isn't "Getting Smarter" (The Memory Gap)
True "Learning" in AI requires a Memory Store (usually a Vector Database like Pinecone or a simple "Learned Patterns" table).

The Gap: When you click "Optimize Agent" in your "AI Learning Center," the app probably just runs a prompt. It doesn't actually store what it learned in a way that the next Generation call can read.

The Platinum Upgrade:

The Style Ledger: Every time a user "Accepts" an article, save the successful patterns (e.g., "User likes bullet points for costs") to a team_preferences table.

The Pre-Generation Injection: Before writing an article, the app should query the team_preferences. It then injects a "Memory Block" into the prompt: "In the past, this user has rejected articles without local SEO keywords. Ensure 'Boston' is mentioned 3 times."

🚀 Architect's Suggestions to Better the Whole App
If we want this to be a "Platinum" Enterprise tool, the app should be proactive, not reactive. Here is how we bridge the UX and Intelligence gaps:

1. The "Actionable Dashboard"
Your current dashboard is just stats. It should be a Command Center.

Gap: You see "142 Total Posts" but have to hunt to find what to do next.

Fix: Add a "Priority Queue" that says: "3 articles were generated yesterday but haven't been pushed to Social Media yet. Click here to batch-post."

2. Global "Health Score"
Every article should have a "Platinum Score" (0–100).

Fix: Use your new Guardian Agent to calculate this score based on SEO, Persona match, and Formatting. If an article is under 80, the "View" button turns red, signaling it needs a "Quick Fix" (the fix-hyperlinks route we built).

3. Integrated Sidebar (The "Thread" of the App)
Get rid of the "Home" grid for navigation.

Fix: Implement a persistent Contextual Sidebar. If I'm looking at an article, the sidebar should show the Media Library images that fit that article's topic automatically. No searching required.
