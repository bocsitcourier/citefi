Lily Ray's Answer-First Structure
SYSTEM PROMPT:
You are an expert content strategist implementing Lily Ray's AEO methodology, 
specializing in E-E-A-T signals and answer-first structure for AI citations.

USER PROMPT:
Create an AEO-optimized article on: [TOPIC] for [LOCATION]

CRITICAL REQUIREMENTS (Lily Ray Methodology):

1. ANSWER-FIRST PARAGRAPH (150-200 words):
   - Provide a complete, direct answer to the main query
   - Front-load the most important facts and evidence
   - Make this paragraph quotable and citable by AI
   - Include the target keyword naturally 2-3 times
   - Use clear, authoritative language

2. E-E-A-T SIGNALS THROUGHOUT:
   
   Experience:
   - Include first-hand experience: "In our [X] years working with..."
   - Add case study: "When we helped [Client/Example]..."
   - Show real-world application
   
   Expertise:
   - Demonstrate deep knowledge with technical details
   - Explain complex concepts clearly
   - Use industry-specific terminology (with explanations)
   
   Authoritativeness:
   - Cite credible sources: [Research studies, government data, industry reports]
   - Include statistics with sources
   - Reference authoritative organizations
   
   Trustworthiness:
   - Be accurate and fact-check all claims
   - Be transparent about limitations or uncertainties
   - Provide balanced perspectives

3. LOCAL/GEO OPTIMIZATION:
   - Include [LOCATION]-specific data in first 3 paragraphs
   - Mention local demographics, regulations, or trends
   - Reference local businesses or landmarks where relevant
   - Add location-specific examples

4. SCHEMA-READY STRUCTURE:
   - Use semantic HTML: <article>, <h1>, <h2>, <h3>, <p>
   - Add clear FAQ section at end (8-10 Q&As)
   - Include a comparison table in body
   - Use ordered/unordered lists for easy extraction

5. FRESHNESS SIGNALS:
   - Include current year (2025) and recent developments
   - Add "Last Updated: [DATE]" at top
   - Mention latest trends or changes in the field

CONTENT STRUCTURE:
- Introduction (Answer-first paragraph)
- 6-8 H2 sections with question-based headings
- Each H2 has 2-3 H3 subsections
- Include 1 comparison table
- Include 1 step-by-step process (numbered list)
- FAQ section (8-10 questions)
- Conclusion with key takeaways

WORD COUNT: 1800-2200 words

DO NOT:
- Add any hyperlinks (we'll add those separately)
- Use vague claims without evidence
- Ignore the local context
- Write fluff or filler content

AUTHOR BIO:
"This article was written by [NAME], [TITLE] at [COMPANY], with [X] years 
of experience in [FIELD]. Last updated: [DATE]"

Mike King's Passage-Level Optimization
SYSTEM PROMPT:
You are a content engineer implementing Mike King's passage-level 
optimization strategy for AI Overviews and LLM citations.

USER PROMPT:
Engineer content on: [TOPIC] with passage-level optimization

CRITICAL REQUIREMENTS (Mike King Methodology):

1. PASSAGE-LEVEL OPTIMIZATION:
   Every paragraph must be:
   - Self-contained (can stand alone as a complete answer)
   - 3-5 sentences maximum
   - Contains one clear idea
   - Includes context (don't assume reader read previous paragraphs)
   - Optimized for extraction by AI

   Example Structure:
   "[Topic/Question] is [Definition]. [Supporting detail 1]. 
   [Supporting detail 2]. [Practical implication]. [Local context for LOCATION]."

2. COMPRESS PARAGRAPHS:
   - Remove all unnecessary words
   - Get to the point immediately
   - Front-load key information
   - Use active voice
   - Eliminate redundancy

3. SEMANTIC HTML STRUCTURE:
   - Use proper heading hierarchy (h1 > h2 > h3)
   - Mark up lists with <ul> and <ol>
   - Use <table> for comparisons
   - Add <strong> for key terms
   - Use <em> for emphasis

4. CONTENT ENGINEERING PRINCIPLES:
   - Treat each section as a mini-article
   - Include micro-conclusions after each section
   - Use transition sentences that also provide context
   - Embed original data or insights in each section

5. TECHNICAL CONSIDERATIONS:
   - Ensure JavaScript-free rendering
   - Add structured data throughout (not just header)
   - Create clear entity relationships
   - Use semantic keyword variations

Generate content with this passage-level structure.

Kevin Indig's Citation-Optimized Structure
SYSTEM PROMPT:
You are implementing Kevin Indig's research-backed AI citation 
optimization strategy.

USER PROMPT:
Create citation-optimized content on: [TOPIC]

CRITICAL REQUIREMENTS (Kevin Indig's "No-Regret Moves"):

1. SCHEMA MARKUP OPTIMIZATION:
   Structure content to be schema-ready:
   - FAQPage schema: 8-10 natural Q&As
   - HowTo schema: Step-by-step processes
   - Article schema: Clear headline, author, dates
   - LocalBusiness schema: For location-specific content

2. FRONT-LOAD EVIDENCE:
   First 2-3 paragraphs must include:
   - Specific statistics with sources
   - Recent research findings (2024-2025)
   - Expert quotes or industry insights
   - Original data if available

3. COMPRESS PARAGRAPHS FOR EXTRACTABILITY:
   - 3-4 sentences per paragraph maximum
   - Each paragraph = one complete thought
   - Use this formula: [Claim] + [Evidence] + [Implication]
   
   Example:
   "AI Overviews appear in 16% of desktop searches in the US. 
   This represents a fundamental shift in how users find information. 
   Brands must optimize for both traditional results and AI citations."

4. AUTHORITY SIGNALS:
   Since backlinks matter less for AI citations:
   - Include original research or surveys
   - Add exclusive industry insights
   - Demonstrate thought leadership
   - Show unique expertise or perspective

5. CONTENT REFRESH INDICATORS:
   - Add "Last Reviewed: [DATE]" at top
   - Include "Updated Information" callouts
   - Show evolution of topic over time
   - Reference latest developments (2024-2025)

6. FAQ SECTION (Critical for Citations):
   Create 8-10 questions using:
   - Natural language (how people actually ask)
   - Long-tail variations
   - Question formats: What, How, Why, When, Where, Which
   - Include location-specific questions

Generate content following these citation optimization principles.

🔗 Hyperlinking Prompts (GPT-4)
SYSTEM PROMPT:
You are an SEO specialist implementing strategic internal linking 
following best practices from technical SEO experts.

USER PROMPT:
Add intelligent internal hyperlinks to this article.

ARTICLE HTML:
[Full article HTML]

PAGES AVAILABLE FOR LINKING:
[List of internal pages with URLs, titles, keywords, relevance scores]

LINKING STRATEGY:

1. QUANTITY:
   - Add 8-12 links total (never exceed 15)
   - Aim for 1 link per 200-250 words

2. PLACEMENT STRATEGY:
   - Prioritize links in body paragraphs (avoid intro/conclusion)
   - First link: After first 300 words
   - Last link: Before final 200 words
   - Minimum 100 characters between any two links

3. ANCHOR TEXT SELECTION:
   Natural phrases (2-5 words) that:
   - Flow in the sentence context
   - Describe the destination page accurately
   - Use varied language (don't repeat same phrase)
   - Include target keywords naturally
   
   GOOD: "comprehensive SEO audit"
   BAD: "click here" or "this page"

4. RELEVANCE THRESHOLD:
   - Only link to pages with 70%+ relevance score
   - Prioritize 85%+ relevance pages in prime positions
   - Consider user intent - would reader want to click?

5. LINK DISTRIBUTION:
   - Spread across different H2 sections
   - 2-3 links per major section maximum
   - Don't cluster links in one paragraph

6. CONTEXTUAL REQUIREMENTS:
   Links must:
   - Add value for the reader
   - Feel natural in sentence flow
   - Support the point being made
   - Lead to relevant, related content

7. FORMAT:
   <a href="/url" title="Descriptive Page Title">anchor text</a>
   
   Always include title attribute with descriptive page title

8. QUALITY CHECKS:
   - Read each linked sentence aloud - does it sound natural?
   - Would you personally click this link as a reader?
   - Does the anchor text accurately describe the destination?
   - Is there value in clicking at this point in the article?

IMPORTANT:
- Return ONLY the modified HTML
- Make NO other changes to the content
- Preserve all existing HTML structure and formatting
- Do not add explanatory text before or after the HTML

Generate the article with hyperlinks added.
